### ARCH BUILD SYSTEM ###

	ABS позволяет автоматизировать некоторые задачи, связанные 
	с компиляцией из исходников: 
	
		- Компиляция или перекомпиляция пакета

		- Сборка из исходных кодов и установка программ, для которых
		еще нет официального пакета 

		- Изменение существующих пакетов под свои нужды 
		
		- Перестройка всей системы с использованием новых флагов
		компиляции

		- Чистая сборка и установка собственного ядра 

		- Создание модулей для собственного ядра

		- Компиляция и установка новой/старой/beta/devel версии Arch
		пакета с помощью редактирования номера версии в PKGBUILD
			
	
	Получение PKGBUILD from git
		
		# Установить пакет ASP, инструмент для скачивания исходных файлов
		пакета из Arch Linux через Git-интерфейс
			
			@ sudo pacman -S asp  
		
		# Выполнить сверку пакета из репозитория

			@ asp checkout name_of_package/package's name
	
		# Для обновления клонированног git репозитория необходимо
		перейти в него последовательно выполнить команды
			
			@ asp update/git pull

		# Если необходимо просто получить копию снимка (snapshot)
		текущего PKGBUILD для конкретного пакета
			
			@ asp export name_of_package/package's name

	Сборка пакета 
		
		# Перед сборкой пакета из файла PKGBUILD настроить makepkg в
		соответствии с указаниями из раздела

		PKGBUILD:

		OPTIONS += (debug !strip)
			
			# Эти настройки будут включать компиляцию с отладочными
			символами и отключат их удаление из исполняемых файлов
			
			
		options = (debug !strip)

			# Для изменения параметров отдельного пакета нужно
			изменить его PKGBUILD 

		# Некоторые пакеты к примеру glibc, все равно удалаяет
		отладочные символы PKGBUILD:
			
			strip $STRIP_BINARIES

			strip $STRIP_STATIC
